# Security Policy

## Supported Versions

Security fixes are applied to the **latest release** only.
Older versions are not backported.

| Version | Supported |
| ------- | --------- |
| Latest  | ✅        |
| Older   | ❌        |

## Reporting a Vulnerability

Please **do not** open a public GitHub issue for security vulnerabilities.

Use GitHub's private vulnerability reporting feature instead:

1. Go to the [Security Advisories](https://github.com/troykelly/homeassistant-emby/security/advisories) page.
2. Click **Report a vulnerability**.
3. Fill in the details — include steps to reproduce, affected versions, and potential impact.

You will receive an acknowledgement within **72 hours** and a status update within **7 days**.

## Out of Scope

The following are outside the scope of this security policy:

- Vulnerabilities in the **Emby Media Server** itself — report those to [Emby](https://emby.media).
- Vulnerabilities in **Home Assistant core** — report those to the [HA security team](https://www.home-assistant.io/security/).
- Issues that require physical access to the Home Assistant host.

## Disclosure Policy

Once a fix is released, a [GitHub Security Advisory](https://github.com/troykelly/homeassistant-emby/security/advisories) will be published with full details.
